# WORK LABS Site Strategy

## Core Point Of View

WORK LABS should present itself as one intelligent system with several clear expressions of value.

- `About` explains the lineage, philosophy, and long-view ambition.
- `CI²` explains the operating thesis.
- `Portfolio` proves the standard.
- `Brand Foundry` shows productized strategic IP.
- `Consulting` explains how the thinking gets applied in live engagements.
- `Contact` routes inquiries by intent.

The site should feel founder-led but not founder-dependent. Cabell is the source of the philosophy and the standard, not the sole container of the brand.

## Client Answers Incorporated

- Primary buyers: broad by design. The site needs to speak to companies seeking branding, identity, packaging, strategy, project-specific work, and consulting support.
- Priority over the next 12 months: CI² is the spine through everything. The site should help WORK LABS own that thinking.
- Portfolio job: impress, qualify, and convert.
- Foundry visibility: five hero brands can be public; deeper inventory should be available by request, potentially organized by category.
- Cabell's role: founder and creative director, building the foundation for future growth.
- Lead goal: request for a call, stronger engagement, and stronger SEO.
- CI² definition: Collaborative Intelligence is the operating philosophy of WORK LABS.

## Home Page Job

The home page should quickly answer:

1. What is WORK LABS?
2. Why is it different?
3. What are the main doors in?
4. Why should someone trust it?
5. What should they do next?

## Messaging Guidance

- Reduce first-person language.
- Turn manifesto language into commercial clarity.
- Use the founder story as origin, not as the structure of every page.
- Treat the "firsts" as lineage and evidence, not a brag roll.
- Make CI² feel like the system underneath every offer, not a side project.

## Suggested CTA Logic

- Home: request a call or choose a path
- About: explore CI² or contact
- CI²: inquire about partnership, licensing, speaking, or consulting
- Portfolio: start a conversation
- Brand Foundry: request access or inquiry
- Consulting: book a strategic conversation
- Contact: self-select by inquiry type

## Remaining Decisions

1. Which five Foundry hero brands should be public first?
2. Which categories should appear for request-only Foundry inventory?
3. Which portfolio case studies best support the current CI² positioning?
4. What exact contact flow should power "request a call" conversions?
5. Should SEO landing pages eventually be created for services like identity, packaging, and strategy?

## Build Notes

- Astro is a good fit because the site is content-led and page-driven.
- Tailwind keeps the design system fast to evolve while messaging is still changing.
- Content is centralized in `src/data/site.ts` so page direction can be revised before a CMS is introduced.
